📐 Normalization
✅ 1NF — First Normal Form
  • All columns have atomic values
  • No repeating groups
  • Each column has a single value
  • Every table has a primary key
✅ 2NF — Second Normal Form
  • Satisfies 1NF
  • No partial dependencies
  • All non-key columns depend on full primary key
  • Separate tables for products and categories
✅ 3NF — Third Normal Form
  • Satisfies 2NF
  • No transitive dependencies
  • Customer info stored only in customers table
  • Product info stored only in products table
💡 Key Design Decisions
Decision Reason
price stored in order_items separately Preserves historical price at time of purchase
IDENTITY columns for all PKs Auto-increment prevents duplicate key errors
Foreign keys throughout Enforces referential integrity across tables
password stored as SHA-256 hash Security — plain text passwords never stored
role column in customers Differentiates admin and user access levels
Separate shipping table Tracks delivery independently from orders
